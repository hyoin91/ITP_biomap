1. 시스템 요구사항(ECR)
◾운영서버 구성도
○ 플랫폼의 원활한 운영과 관리자·사용자의 편의성 증대를 위해 타 시스템 연계·연동 
  - SMS 시스템 연계 : 회원가입·탈퇴, 비밀번호 초기화 등 주요 서비스에 문자발송 기능 개발
  - 본인인증 시스템 연계 : 회원가입·탈퇴, 비밀번호 초기화 등 주요 서비스에 본인인증 기능 개발
  - 지리정보 API 연계 : 인천 바이오 유관기관 위치정보를 오픈 API를 이용하여 지도에 위치표시
  - SNS 로그인 API, 우편번호 API 연계 : 회원가입 시 SNS로그인과 기관 주소입력을 위해 API 연계 등

2. 기능 요구사항(SFR)
○ 전자정부 표준프레임워크(eGovFramework) v3.5 이상 적용
○ 전자정부 표준프레임워크 3.5이상 인증 CMS도입
  - 5개소 이상 납품실적 제출
○ 웹 표준, 웹 호환성, 웹 접근성, 개발 보안을 준수하여 개발
○ 플랫폼 내 모든 정보는 외부 검색포털 검색 최적화 
○ 다양한 기기에서 최적의 정보서비스를 제공할 수 있는 반응형 웹 기반의 플랫폼 구축
○ 시큐어 코딩을 필수적용하여 개발 시 보안 약점을 제거해야 함
○ 다양한 브라우저에서 서비스가 가능(호환성 준수)
○ HTML5 표준으로 제작 및 non-ActiveX 실현
○ 장애인 및 노약자 등의 정보 소외계층이 이용하는 데 불편함이 없는 메뉴 및 디자인으로 구성

◾디자인 시안, 웹 스타일가이드(Web Style Guide)
○ 인천 바이오 맵 디자인 개발
  - 최신 트랜드를 적용하여 인천 바이오 맵에 부합하는 WI(Web Identity)를 정립하고 전체적으로 일관되게 적용
  - 아이콘, 정보구조, 컬러 등의 디자인 스타일 가이드를 확립하고 플랫폼에 적용
  - 인천 바이오 맵 온라인 플랫폼 메인페이지·콘텐츠페이지 ·게시판 프로그램 등 디자인 제작
 - 메인 시안 2개·카테고리 16개 이내, 전체 컨텐츠 50쪽 내외 제작

◾ 개발 및 디자인 소스
○ 인천 바이오 맵 온라인 플랫폼에 반응형 웹 기술을 적용하여 PC와 스마트기기로 접속하는 모든 사용자에 동일 서비스제공
○ 통합관리자 기능 구현으로 한 번의 콘텐츠 수정으로 PC와 스마트기기 콘텐츠 수정이 가능하도록 구현
○ 플랫폼에서 제공하는 콘텐츠 레이아웃 등은 접속기기의 특성에 따라 자동으로 변경 가능하도록 UI구성
○ W3C 웹 표준 준수로 멀티 브라우저(5종 이상) 지원
○ 「모바일 전자정부 서비스 관리 지침」준수

◾ 운영자(관리자) 매뉴얼1
○ 관리자는 게시판을 생성할 수 있어야 하고 게시판 접근에 대한 권한 설정 기능.
○ 생성된 게시판은 수정 및 삭제가 가능하여야 함.
○ 게시 글은 권한 있는 관리자·사용자만 글을 쓸 수 있도록 하고 게시 글의 수정·삭제 기능 제공
○ 게시 글은 공지기능을 제공하여 중요한 사항에 대해서는 게시판 상단에 위치 할 수 있도록 함.
○ 게시판 유형은 공지형/답글형/갤러리형/링크형 등 플랫폼에서 필요로 하는 유형을 제공.
○ 유사 유형의 게시판간 게시글 이관이 가능하여야 함.
○ 웹 에디터 기능을 제공하고 첨부이미지에 대해서는 대체텍스트입력 기능을 제공해야함.
○ 초급사용자도 쉽게 운영할 수 있도록 사용자 중심의 UI설계
○ 사용자 및 담당자에 따라 게시판 및 메뉴 등에 대한 접근권한 설정기능 제공
  - 조직관리, 관리자 등급관리
  - ID별, 메뉴별(콘텐츠별) 작업권한 관리
  - 추가 메뉴에 대해서도 권한관리 지원
○ 관리자의 모든 행동을 로그로 남겨야 하며 열람 가능해야 함
  - 접속기록 로그(접속IP, ID, 사용자명, 기관, 접속일시 등)
  - 권한 부여, 권한 변경 로그
  - 조회․수정․삭제 및 일체 행위 로그 저장 및 열람기능
  - 로그파일 다운로드 지원
○ 플랫폼내 메뉴순서 및 메뉴이름 변경기능
○ 관리자 메뉴 검색기능 제공
○ 관리자 페이지 접속가능 IP 관리기능
○ 관리자 ID 사용이력 로그 관리 기능 개발
○ 환경정보 및 운영관련 정보를 코드로 정의하여 DB화
○ 관리자 페이지 접속 시 SSL 적용
○ 관리자 페이지는 별도 도메인 URL을 사용하고 접근 통제 및 IP 주소 기반으로 적용
○ 관리자 페이지 로그온 기록 유지 및 장시간 미사용 시 자동 로그아웃
○ 일간, 월간, 연간으로 홈페이지 접속자 수 및 접속자 수 변경 흐름 모니터링 기능 제공
○ GPS 정보를 입력하는 메뉴를 만들고 각 시설 등의 위치정보를 입력
○ 통합 게시판 관리 기능 연계
○ 메뉴별 콘텐츠 관리기능 및 백업·복구 기능제공
  - 콘텐츠 추가·수정·삭제 및 버전별 복구기능

3. 인터페이스 요구사항
○ 타 시스템과의 연계 항목
  - SMS 문자발송 시스템 연계
  - 공공데이터: 공공데이터포탈(주소검색, 날씨 등)
  - 휴대폰 인증 API연계
  - SNS로그인 API연계
  - 공개 Open API 지도서비스 연계

○ 최신 트렌드를 반영한 Style, UI 적용
○ 기능별 일관성 있는 인터페이스 설계 및 구현
○ 콘텐츠로 이동하기 편리하도록 콘텐츠는 가능한 3depth 이내로 구성해야 함
○ 자료 삭제 혹은 입력 완료 등과 같이 사용자의 수행활동에 대한 메시지를 제공해야 함
○ 메인화면과 서브메뉴 간에 사용자 인터페이스의 일관성을 유지해야 함
○ 페이지별 콘텐츠 영역 상단에 페이지에 대한 안내메시지로 그 내용을 이해하기 쉽게 설명
○ Alert 메시지를 통해 기능에 대한 알림, 에러 등 구체적으로 설명 제공
○ Popup, Alert 등 시스템에서 제공하는 메시지는 일관성 있는 스타일과 폼을 사용
○ 정보 입력/등록/신청할 때 삭제나 초기화 버튼 클릭 시 Confirm 메시지 제공
○ 화면마다 메인페이지로 쉽게 이동할 수 있도록 표준화 링크 제공
